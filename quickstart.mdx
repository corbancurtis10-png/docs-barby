This guide walks you through everything you need to go from a blank project to a working animation in under five minutes. You will place the EZAnimation ModuleScript, require it in a script, initialize the service with your animation IDs, and make your first `play` and `stop` calls.

## Complete Working Example

Here is the full script combining every step above into a single, copy-pasteable example. Place this in a `LocalScript` inside `StarterCharacterScripts`.

```lua
-- LocalScript inside StarterCharacterScripts
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Players           = game:GetService("Players")

-- 1. Require EZAnimation
local AnimationService = require(ReplicatedStorage.EZAnimation)

-- 2. Initialize with your animation list
local anim = AnimationService.init({
    List = {
        idle   = "rbxassetid://1234567890",
        run    = "rbxassetid://0987654321",
        attack = "rbxassetid://1122334455",
    }
})

-- 3. Grab the local player's character
local player    = Players.LocalPlayer
local character = player.Character or player.CharacterAdded:Wait()

-- 4. Play the idle animation (looped, Idle priority, 0.2 s fade-in)
anim:play(character, "idle", "Idle", true, 0.2)
print("Playing idle animation")

-- 5. After 5 seconds, stop the idle animation
task.delay(5, function()
    anim:stop(character, "idle", 0.3)
    print("Stopped idle animation")
end)
```

<Warning>
  Always make sure the `character` model is fully loaded before calling `play`. Using `player.CharacterAdded:Wait()` (as shown above) ensures the character exists before you attempt to animate it.
</Warning>
